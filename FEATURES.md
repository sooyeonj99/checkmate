## 2. 사용자별 사용 흐름

### 👤 개인 사용자 (personal)

```
회원가입 → 이메일 인증 → 로그인
  └─ 계약서 업로드 (파일/카메라)
       └─ 개인정보 마스킹 (선택)
            └─ AI 분석 (자동)
                 └─ 결과 확인 (위험도 점수 + 조항별 분석 + 수정 제안)
                      ├─ 대시보드에 저장
                      ├─ 만료일 설정 → 만료 7일 전 알림
                      ├─ PDF 리포트 다운로드
                      └─ AI 챗봇으로 계약서 질문
```

**핵심 기능:**
- 계약서 분석, 저장, 챗봇
- AI 생성기 (새 계약서 작성)
- 법령 추적 (준비중)

---

### 🏢 기업 사용자 (enterprise)

```
회원가입(기업) → 팀 구성
  ├─ 팀원 초대 (이메일/SMS)
  └─ 기업 계약서 분석
       ├─ 일괄 분석 (10개 동시)
       ├─ 계약서 비교 분석 (A vs B)
       ├─ 전자서명 요청 → 상대방 이메일로 링크 발송
       │    └─ 상대방: 링크 클릭 → 서명 → 완료 알림
       ├─ 서명 기록 관리 (보낸/받은 서명)
       └─ 팀 내 계약서 공유 및 분석 통계
```

**핵심 기능:**
- 일괄 분석, 비교 분석
- 전자서명 요청/관리
- 팀원 협업, 통계 대시보드

---

### 🏪 프랜차이즈 본사 (franchisor)

```
프랜차이즈 본사 계정 → 가맹점주 초대
  └─ 표준 계약서 템플릿 작성
       └─ 가맹점에 계약서 배포
            ├─ 각 가맹점: 계약서 수신 → 검토 → 서명
            ├─ 본사: 전체 가맹점 서명 현황 조회
            └─ 만료 계약 일괄 갱신 관리
```

**핵심 기능:**
- 템플릿 편집기 (표준 계약서 작성)
- 프랜차이즈 계약 배포
- 가맹점 계약 현황 모니터링

---

### 🏬 가맹점주 (franchisee)

```
초대 링크 클릭 → 가맹점주 계정 생성
  └─ 본사에서 배포된 계약서 수신 알림
       └─ 계약서 AI 분석 (자동 실행)
            ├─ 분석 결과 확인 (위험 조항 강조)
            ├─ AI 챗봇으로 질문
            └─ 동의 후 전자서명
```

**핵심 기능:**
- 본사 배포 계약서 자동 분석
- 전자서명

## 4. B2B API 사용법

```bash
# 계약서 업로드
curl -X POST https://101.79.25.139/api/v1/contracts/upload \
  -H "X-Api-Key: <발급된 키>" \
  -F "file=@contract.pdf"

# 분석 실행
curl -X POST https://101.79.25.139/api/v1/contracts/{contract_id}/analyze \
  -H "X-Api-Key: <발급된 키>"
```
